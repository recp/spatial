# 2D variant

For 2D engines (Box2D, Rapier2D, custom 2D work). Parallel to the 3D
types but never mixed with them.

## spatial_rot2_t

```c
typedef struct spatial_rot2_t {
  float c;   /* cos(angle) */
  float s;   /* sin(angle) */
} spatial_rot2_t;
```

Matches Box2D `b2Rot` exactly. Stored as cos/sin to avoid repeated
trig during composition.

### Invariants

- `c*c + s*s == 1` (± epsilon).
- `SPATIAL_ROT2_IDENTITY = { 1, 0 }`.

## spatial_transform2_t

```c
typedef struct spatial_transform2_t {
  vec2        position;   /* 8 bytes  */
  spatial_rot2_t rotation;   /* 8 bytes  */
} spatial_transform2_t;      /* 16 bytes */
```

Matches `b2Transform`. No scale on the body transform (same reasoning
as 3D).

## spatial_pose2_t

```c
typedef struct spatial_pose2_t {
  vec2        position;
  spatial_rot2_t rotation;
  vec2        scale;
} spatial_pose2_t;
```

Graphics / editor variant with 2D scale.

## Composition

```
world.position = parent.position
               + rotate2(parent.rotation, parent.scale * local.position)
world.rotation = rot2_mul(parent.rotation, local.rotation)
world.scale    = parent.scale * local.scale
```

Where `rotate2` applies a `spatial_rot2_t` to a `vec2`, and `rot2_mul`
composes two rotations (equivalent to angle addition).

## Interop with 3D

A 2D space and a 3D space are separate `spatial_space_t` instances.
No implicit promotion. Callers that need 2D poses in a 3D world
**MUST** explicitly embed:

```c
void spatial_transform2_to_3_xy(const spatial_transform2_t *in,
                             spatial_transform_t *out);  /* z=0, axis=+Z */
```

## Conformance

An implementation **MAY** omit the 2D variant. A full implementation
**MUST** support both and document which 2D engines it has been
tested against.
